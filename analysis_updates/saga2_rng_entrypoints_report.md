# SaGa2 RNG entrypoints report

## 対象

- `rom/SaGa 2 - Hihou Densetsu (J) (V1.1).gb`

## 目的

- `00:0258` の seed 初期化がどう到達されるかを確認する
- `00:0440` 周辺の高位入口 / wrapper を整理する

## 1. `00:0258` は direct call ではなく起動シーケンスの fallthrough

`00:0200` から始まる初期化コードを読むと、`00:0258` へ向けた `CALL/JP` はなく、
RAM clear や初期セットアップの直後にそのまま実行が流れ込む。

### 周辺の流れ

- `00:0200`
  - 割り込み停止
  - レジスタ初期化
  - RAM/HRAM クリア
- `00:0258`
  - `random_seeds ($C0A0)` 64 byte 初期化
- `00:0280`
  - そのまま次のシステム初期化へ継続

### 判断

`00:0258` は独立 API というより、**起動時の seed 初期化ブロック** とみるのが自然。

## 2. `00:0440` の上位入口は `00:0469`

`00:0440` そのものへの direct call/jp は未検出だったが、`00:0469` には direct xref があった。

### direct xref

- `JP 00:0469` at `00:0171`
- `CALL 00:0469` at `00:0494`
- `CALL 00:0469` at `00:0CC8`

### 解釈

`00:0440` は公開入口ではなく、`00:0469` から入る **内部 RNG 消費本体** の可能性が高い。

## 3. `00:0171` は jump table 由来の入口候補

`00:0160-017F` には

```text
C3 90 03
C3 A6 03
...
C3 69 04
C3 9D 04
...
```

のような `JP` 列が並ぶ。

このため `00:0171 -> JP 00:0469` は、固定呼び出しというより
**共通 dispatch table の 1 エントリ** である可能性が高い。

## 4. `00:0493` は `00:0469` を包む thin wrapper

### バイト列

```text
CD 69 04
CD 8F 06
F0 8A
C9
```

### 擬似コード

```text
call 00:0469
call 00:068F
A = [$FF8A]
ret
```

### 判断

- RNG 消費後に別の状態更新 / 同期待ちを挟む wrapper
- 戻り値は `FF8A` 経由で返している可能性が高い

## 5. `00:0CC8` は `00:0469` を使うループ型 wrapper

### 周辺バイト列

```text
CD A6 04
CD 8F 06
CD 69 04
F0 8A
A7
28 F5
C3 A6 04
```

### 擬似コード

```text
loop:
  call 00:04A6? / pre-step
  call 00:068F
  call 00:0469
  A = [$FF8A]
  if A == 0: repeat
  jump 00:04A6?
```

### 判断

これは `00:0469` を単発で呼ぶのではなく、**条件が整うまで回す高位ルーチン** 候補。

正確な意味はまだ未確定だが、少なくとも RNG 本体より一段上の制御層に属する。

## 6. 現時点の整理

### 確度が高いこと

- `00:0258` は起動時 seed 初期化ブロック
- `00:0440` は `random_seeds[index]` を進めて `data_rng` を読む内部本体
- `00:0469` は `00:0440` を含む高位入口候補
- `00:0493` と `00:0CC8` は `00:0469` を利用する wrapper / 制御層候補

### まだ未確定なこと

- `00:0469` の厳密な関数境界
- `FF89` / `FF8A` の意味
- `00:068F` の役割
- `00:0CC8` ループの用途

## 次の一手

1. `00:068F` と `FF89/FF8A` の役割を切る
2. `00:0493` / `00:0CC8` の呼び出し元をさらに上へ辿る
3. battle 側で RNG がどの入口から使われるかを分類する
