# Artifact Boundaries

## Status

working_model

## Purpose

This repository should stay small enough for future GPT/Codex sessions to load the right context quickly. The main rule is to commit durable analysis inputs and small reviewed fixtures, while keeping bulky or regenerable extraction products local.

## Committed Canonical / Durable

Commit these when they are reviewed and useful as entrypoints:

- topic or scene summaries under `docs/`
- reverse-engineering notes under `analysis_updates/`
- compact binary-analysis packages under `docs/binary_analysis/`
- small machine-readable artifacts under `build/binary_analysis/`
- scenario fixtures under `docs/scene_packs/*/scenarios/`
- source tools under `tools/`
- generated `INDEX.md` files

## Local / Regenerable

Keep these out of normal commits:

- original ROM files
- local ROM extraction previews under `build/rom_extracts/`
- PNG previews generated during one-off visual searches
- `.png.import` files generated by Godot
- local Godot, .NET, NuGet, and editor caches
- exported app bundles and local run logs

## Promotion Criteria

Promote a local artifact only when all are true:

- it answers a current scene-pack question
- it is small enough to review in a diff
- the source tool and source ROM assumptions are documented
- the relevant scene pack links to it
- it does not contain more ROM-derived bytes than needed for the analysis case

If an artifact is large or easy to regenerate, commit the tool and document the command instead.

## Current Build Split

- `build/binary_analysis/`: committed, durable derived analysis data.
- `build/rom_extracts/`: local generated-output namespace; only `INDEX.md` signposts are tracked.
